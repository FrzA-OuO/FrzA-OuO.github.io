    计算机网络复习笔记

    发表于 2020-01-09 分类于 笔记 本文字数： 15k 阅读时长 ≈ 28 分钟

    平时课程与期末复习时的总结笔记。

    CH1 介绍

    交换技术

    Circuit switching 电路交换

    电路交换中，沿着端系统通信路径，为端系统之间通信所提供的资源在通信会话中会被预留。比如电话网络。这是一个真正意义上的连接，被称为一条电路circuit。能够确保以恒定速率向接收方传送数据。

    Multiplexing 多路复用技术 在电路交换网络中

    frequency-division multiplexing FDM 频分多路复用：频段的宽度称为带宽bandwidth

    time-division multiplexing TDM 时分多路复用：时间被分为定区间的帧，每一帧又被分为固定数量的time slot时隙。网络为一个连接在每个帧内指定一个时隙。一条电路的传输速率为一个time slot 中的bit数乘以该frame的速率。

    以上两种复用技术的资源利用率提升有限，对于重载网络效果更好，对轻载网络资源利用率低，需要知道使用人数。

    packet switching 包交换

    不预留资源。因特网是包交换网络。

    应用程序要交换message报文。报文能包含协议设计者需要的任何东西。源主机将长报文划分成较小的packet包。这些包/分组通过通信链路和packet switch分组交换机传送。包以链路的最大传输速率在通信链路上传输。

    多数分组交换机采用store-and-forward transmission 存储转发传输机制：在交换机能够开始向输出链路传输该包的第一个bit前必须收到完整的packet。因而引入了存储转发延迟packet还会有queue delay排队时延。网络过于拥塞会发生packet lost丢包。

    statistical multiplexing 统计多路复用：按需共享资源（谁需要，谁使用）

    包交换网络的端到端delay不可预测，但提供了更好的带宽共享，比电路交换网络更简单。

    延迟和丢包

    nodal processing delay 节点处理延迟 检查头部，差错检测等，通常为微秒级。

    queuing delay 排队延迟 取决于网络拥塞程度，通常是毫秒到微秒级

    transmission delay 传输延迟 （又称存储转发延迟），为包大小/链路传输速率

    propagation delay 传播延迟 传播距离/介质传播速率

    throughput 吞吐量：每秒能够传输的数据量。最大吞吐量：一台路由器能够转发包的最大速率。

    两个进程在一条网络路径上进行通信会话时，可用吞吐量指发送进程能够向接收进程交付比特的速率。可用吞吐量会随时间波动。

    丢包

    流量强度traffic intensity La/R 的少量增加会导致时延大量增加。La/R接近于1时，路由器会将到来的包丢弃，即网络拥塞使得路由器到达速率超过转发速率，从而丢包。丢包的根本原因是网络拥塞。

    Protocol Layer协议层次模型

    一个协议层能够用软件、硬件或二者的结合来实现。各层的所有协议称作protocol stack 协议栈

    五层因特网协议栈：应用层、传输层、网络层、链路层、物理层

    七层ISO OSI参考模型：应用层、表示层、会话层、传输层、网络层、链路层、物理层

    packets of information: Message 报文、Segment段、IP datagram 数据报、frame帧、bit比特

    网络构成

    硬件：端系统，链路、设备｛repeater, hub | switch, bridge | Router | Host｝

    软件：操作系统、协议protocol、标准RFC（Request for comments）、IETF（Internet Engineering Task Force）

    Protocol 协议：协议定义了在两个或多个通信实体之间交换的报文格式和次序，以及在报文传输和/或接收或其他事件方面所采取的动作。

    End System 端系统：与因特网相连的计算机等设备，也称作Host主机。

    UTP unshielded twisted pair 非屏蔽双绞线；fiber-optic 光缆；coaxial cable 同轴电缆

    因特网是网络的网络。

    **ISO International Organization for Standardization ** 国际标准化组织

    OSI Open System Interconnection 开放系统互联 模型：将计算机网络分为七层。

    CH2 Application Layer 应用层

    协议数据单元PDU: Message报文

    进行通信的是进程而不是程序，网络应用程序由成对的进程组成，通过网络相互发送报文。进程通过socket套接字这个软件接口发送和接收报文。

    socket 位于应用层和传输层之间，也被称作API Application Programming Interface 应用程序编程接口：应用程序开发者可以控制套接字在应用层端的所有东西

    客户端发起通信，在会话开始时等待联系的是服务器。

    应用程序体系结构application architecture :C/S结构，P2P结构

    Client-server architecture: 客户端-服务器结构

    有一个总是打开的主机，称为server服务器，服务于来自许多其他称作client客户机的主机请求。服务器的地址固定（静态IP），一直在线，客户机之间不相互通信。

    server端有连接池，叫做信源，Client端叫做信宿

    例子：Web, FTP, Telnet, Email

    P2P peer-to-peer architecture 点对点结构

    P2P应用中，一个进程可以即是客户机又是服务器。

    任意间断连接的主机对，称作对等方peer，直接相互通信，不必通过专门的服务器

    例子：文件分发，文件共享，互联网电话，IPTV

    混合模式：服务器用于跟踪用户的IP地址，用户到用户的报文在用户主机之间直接发送。

    P2P具有自扩展性self-scalability，每个peer请求文件产生负载，同时分发文件增加了服务能力。P2P具有高度分布和开方的性质。

    传输层提供了可靠数据传输、吞吐量（带宽）、实时（延迟）、安全性保证。TCP和UDP都无法保证实时性和吞吐量。

    实时音视频能够容忍丢失（loss-tolerant）

    带宽敏感bandwidth-sensitive的应用（多媒体应用，比如直播，网络电话）有吞吐量要求。

    弹性elastic 应用能够充分利用可用带宽，如邮件，文件

    应用层服务

    Web应用和HTTP协议

    WWW World Wide Web 万维网

    HTTP HyperText Transfer Protocol 超文本传输协议，是Web的核心，由两部分程序：客户端程序和服务端程序实现，它们运行在不同端系统中，通过交换HTTP报文进行会话。

    Web page 网页由对象object组成，对象可以是HTML文件，图像，音视频等。多数Web页面由一个基本HTML文件 base HTML file 和一些引用对象referenced object组成。

    URL Uniform Resource Location 统一资源定位符：也被称为网页地址，是因特网上标准的资源的地址。URL由两部分组成：服务器主机名和对象的路径名。Web服务器用于存储对象，每个对象由URL寻址。

    HTTP使用TCP作为其支撑传输层协议。HTTP协议不保存客户机状态信息，因此是一个无状态stateless协议。

    持久连接与非持久连接

    non-persistent connection 非持久连接：每个请求/响应对经一个单独的TCP连接发送

    HTTP1.0 下：客户端发起TCP连接建立请求、客户端发送HTTP请求、服务器响应该请求，封装对象并返回响应、服务器通知TCP断开TCP连接，客户端响应该报文，TCP连接关闭。

    RTT Round-Trip Time往返时间：一个packet从客户端到服务器再回到客户端所花费的时间，其包括传播延迟、排队延迟、分组处理延迟。

    HTML的total response time总响应时间粗略的讲，是两个RTT加上服务器传输文件的时间。（TCP三次握手的前两次是一个RTT、第三次握手加上服务器响应文件是一个RTT）

    非持久连接需要为每一个请求的对象建立和维护一个全新的连接，会给服务器带来严重负担。

    persistent connection持久连接：所有请求和响应使用相同的TCP连接发送。HTTP1.1默认使用流水线方式的持久连接。

    Cookie：HTTP是无状态的，其使用Cookie 让站点跟踪用户。Cookie可以用于标识用户，通过Cookie 可以在无状态的HTTP上建立用户会话层。

    Web Cache，也叫Proxy Server，是能够代表初始服务器来满足HTTP请求的网络实体。Web Cache既是服务器又是客户机，其能够大大减少机构内部网与因特网接入链路上的通信量，从整体上大大降低Internet上的Web流量。

    FTP

    FTP File Transfer Protocol文件传输协议

    采用TCP连接，使用两个并行的TCP连接来传输文件，一个是control connection控制连接，一个是data connection 数据连接。控制连接用于在两个主机之间传输控制信息，数据连接实际传输一个文件。FTP的控制信息是out-of-band的，HTTP是in-band的。

    通常21端口是命令端口，20端口是数据端口。当混入主动/被动模式的概念时，数据端口就有可能不是20了。

    主动模式：客户端任意端口向服务端21端口建控制连接，服务器20端口主动向客户端任意端口+1建数据连接

    被动模式：FTP客户机通过服务器21端口发起用于控制的TCP连接，之后客户端提交PASV命令，服务器开启一个非特权端口并发回给客户端，客户端通过该端口建立数据连接。

    FTP的控制连接是持久的，数据连接是非持久的；FTP会在整个会话期间保留用户状态信息。

    电子邮件

    电子邮件由user agent用户代理、mail server邮件服务器、SMTP 简单邮件传输协议组成。

    SMTP Simple Mail Transfer Protocol 简单邮件传输协议使用TCP。SMTP有两个部分：运行在发送方邮件服务器的客户机端和运行在接收方邮件服务器的服务器端。SMTP默认端口号为25。SMTP一般不使用中间服务器发送邮件，邮件不会在中间服务器停留。SMTP使用持久连接，可以通过一个TCP连接发送多个报文。

    HTTP是pull protocol拉协议，用户从服务器拉取信息；SMTP是push protocol推协议，发送邮件服务器把信息推送到接收服务器。

    MIME Multipurpose Internet Mail Extension 多用途因特网邮件扩展是对SMTP的扩展。用于发送非ASCII内容。

    邮件访问协议：接收方用户代理使用邮件访问协议把邮件服务器上的邮件传送到本地PC。

    • POP3Post Office Protocol version 3 第三版邮局协议 默认端口110

      POP3支持「下载并删除」方式和「下载并保留」方式。POP3服务器不在POP3会话中携带状态信息。

    • IMAP Internet Mail Access Protocol 因特网邮件访问协议 端口 143

      POP3协议没有提供给用户建立远程文件夹及为报文指派文件夹的方法。

      IMAP协议为用户提供了创建文件夹和在文件夹之间移动邮件的命令，提供了在远程文件夹查询邮件的命令。

    邮件发送过程：

    Alice 的邮件通过用户代理经SMTP/MIME(浏览器上是HTTP)发送到Alice的邮件服务器；邮件服务器通过SMTP/MIME发送给Bob的邮件服务器；Bob的用户代理使用POP3/IMAP（或HTTP）收到邮件。

    DNS

    主机可以通过hostname主机名或IP地址识别。

    DNS Domain Name System 域名系统是一个能进行主机名到IP地址转换的目录服务。

    • 是一个由分层的DNS Server实现的分布式数据库
    • 是一个允许主机查询分布式数据库的应用层协议

    DNS运行在UDP上，端口号53。

    除主机名到IP的转换外，DNS还提供了host aliasing主机别名、mail server aliasing邮件服务器别名、load distribution负载分配的功能。

    集中式DNS的一些问题：

    • 单点故障
    • 通信容量有限
    • 远距离带来的高延迟
    • 维护

    DNS的分布式、分层次

    DNS大致分为三种：根DNS服务器、顶级域DNS服务器、权威DNS服务器。

    • Root DNS Server ：互联网上只有13台
    • Top-level Domain (TLD) Server：负责顶级域名和所有国家的顶级域名
    • Authoritative DNS Server：由组织机构提供，如大学和大公司。
    • Local DNS Server：严格上不属于DNS服务器的层次结构；通常由ISP提供。

    查询过程：客户机向本地DNS发送查询报文；本地DNS将之转发到根服务器，根服务器返回负责的顶级域服务器；本地DNS向顶级域服务器发送请求，顶级域服务器返回负责的权威服务器；本地DNS向权威服务器发送请求，权威服务器返回真实IP；本地DNS向客户端返回该DNS.

    注意到，从客户端到本地DNS的查询是recursive query递归查询，后继的本地DNS发出的查询是iterative query 迭代查询.

    请求链中当一个DNS服务器收到DNS回答后，就会将回答中的信息缓存在本地服务器。

    RR Resource Record 资源记录：DNS服务器共同存储着资源记录，其提供了主机名到IP地址的映射。RR是一个包含了(Name, Value, Type, TTL)的四元组

    Type的类型：

    • Type=A，是主机到IP地址的映射
    • Type=NS，则Name是域，Value是知道如何获得该域中地址的权威DNS服务器的主机名。
    • Type=CNAME，则Value是别名为Name的主机的规范主机名。
    • Type=MX，则Value是别名为Name的邮件服务器的规范主机名。

    nslookup命令用于查询DNS

    CH3 Transport Layer 传输层

    传输层位于应用层与网络层之间，其将网络层的在两个端系统间的交付服务扩展到运行在两个不同端系统的应用层进程之间的交付服务。

    传输层协议为运行在不同主机上的应用程序进程之间提供了logic communication 逻辑通信功能。传输层协议是在端系统中而非路由器中实现的。其协议数据单元PDU为Segment报文段。传输层提供两种协议：TCP和UDP。

    TCP和UDP的最基本任务就是将主机之间的交付扩展为端系统进程间的交付，这被称为传输层的 multiplexing 多路复用 和 demultiplexing 多路分解。TCP和UDP都提供了error-detection差错检测功能。

    多路复用与多路分解

    将传输层segment中的数据交付到正确的socket叫做demultiplexing多路分解

    从源主机不同socket中收集数据块并封装上头部信息生成segment，将之传递到网络层的工作叫做multiplexing多路复用

    传输层依靠源端口号和目标端口号区分不同的socket。端口号的范围0-65535，其中0-1023叫做well-known port number.

    • UDP Socket由二元组(Dest. IP 目标IP，目标Port Number)来标识；
    • TCP Socket由四元组(Source 源IP，源Port Number，目标IP，目标Port Number)来标识

    面向连接的，要有单独的Port；无连接的可以只用一个Port

    UDP User Datagram Protocol 用户数据报协议

    提供connectionless 无连接的、unreliable 不可靠数据传输服务。UDP协议不保证报文能够被接受进程接收到，而且可能乱序到达。

    • 由于在发送segment之前，没有握手的连接建立过程，因此称作是无连接的。
    • UDP 提供不可靠的数据传输服务，但UDP Checksum提供了差错检测功能，但并没有差错恢复机制。

    为什么使用UDP:

    • 应用层可以更好地控制发送的数据和发送时间：因为无连接，无拥塞控制
    • 无需连接建立：因此不会引入连接建立的延迟
    • 无连接状态：因此不用维护连接，服务器能支持更多客户机
    • 分组首部的开销小：TCP头有20字节，而UDP头只有8字节

    UDP没有拥塞控制机制。因此可能导致发送方和接收方高丢包率，并挤垮TCP会话。

    UDP头

    UDP头只有4个字段，每个字段2字节：源端口号，目标端口号，segment长度，校验和。

    应用了UDP的协议：

    DNS；RIP；SNMP简单网络管理协议

    RDT 可靠数据传输

    RDT Reliable Data Transfer 可靠数据传输提供给上层一个服务抽象——数据可以通过一条可靠的信道进行传输。可靠信道使得数据不会损坏或丢失、所有数据按照其发送顺序进行传送。其实现通过可靠数据传输协议，但是该协议的下层协议可能是不可靠的，比如TCP是在不可靠的端到端网络层协议IP上实现的可靠数据传输协议。

    FSM Finite-State Machine 有限状态机：其描述了有限的状态之间的转换过程。引起变迁的事件显示在表示变迁的横线上方，事件发生时采取的动作显示在横线下方。初始状态用虚线表示。

    发送方与接收方有各自的FSM

    RDT 1.0 完全可靠信道上的可靠数据传输

    信道完全可靠，接收方不用提供任何反馈信息给发送方，因为不会发生任何差错。

    RDT 2.0 具有比特差错信道上的可靠数据传输

    引入positive acknowledgment肯定确认 ACK(收到了)和negative acknowledgment否定确认 NAK(重复一遍)。

    基于这种重传机制的可靠数据传输协议称作ARQ Automatic Repeat reQuest 自动重传请求协议

    ARQ使用差错检测(checksum)、receiver feedback接收方反馈、retransmission重传实现可靠数据传输

    RDT 2.0中，发送方直到收到反馈后才会发送新数据，故RTD2.0称为stop-and-wait 停等协议

    ！！！RDT 2.0 没有考虑ACK或NAK出错的情况。

    RDT 2.1 引入了sequence number序号

    ACK或NAK出错后，如果重传，则会引入冗余packet，接收方无法判断其是新packet还是一次重传。

    RDT2.1 加入了序号字段，将待发送的分组的序号放在该字段。对于停等协议，添加1bit的序号字段，接收方便可知道发送方是在重传（序号不变）还是发送新packet(序号变化)

    RDT 2.2 具有比特差错信道上无NAK的可靠数据传输

    接收方收到出错的packet后，发送上一次正确接收的ACK；发送方收到对同一个packet的确认两次的ACK（称作duplicate ACK 冗余ACK），便能得知其后面的packet没有被正确接收

    RDT 3.0 具有比特差错的丢包信道上的可靠数据传输

    引入countdown timer倒计数定时器实现在给定的时间过期后若没收到ACK，便中断发送方，重传数据。这会在接收方信道中引入duplicate data packet，其可通过序号解决。

    Pipeline 流水线可靠数据传输协议

    RDT 3.0依旧是停等协议，其性能很糟糕。

    流水线协议允许发送方发送多个分组而无需等待确认。其需要增大序号范围，因为每个传输的packet需要一个唯一序号；需要发送方和接收方都缓存多个packet。其依赖GBN和选择重传。

    GBN Go-Back-N 回退N步

    又被称作sliding-window protocol滑动窗口协议。允许发送方发送多个分组而不等待确认，未确认分组数不能超过N。

    定义最早的未确认分组序号叫做基序号base，将序号最小的待发送分组叫做下一个序号nextseqnum，便可将序号范围分成四部分：已发已确认、已发待确认、可用未发送、不可用(大于等于base+N)。随着协议运行，发送窗口会向后滑动。

    GBN协议中，对序号为n的分组的确认采用cumulative acknowledgment 累积确认的方式，表明接收方正确收到n及n之前的所有packet。（因为一次交付给上层一个packet，若n被交付，则n之前的必定被交付）。

    如果超时，发送方重发所有已发待确认的包。GBN中接收方丢弃所有失序包。

    SR selective repeat 选择重传

    GBN中，单一分组出错可能引发大量分组重传。选择重传通过让发送方仅重传那些它怀疑接收方出错的包来避免不必要的重传。发送方base被确认，则发送窗口移动。

    SR中，接收方将确认一个正确接收的packet而不管是否失序。失序的packet将被缓存直到所有丢失的packet都被收到，这时将一批分组按序交给上层。注意即使已收到，接收方会确认重复的包。

    SR中，窗口长度必须小于等于序号空间大小的一半。

    TCP Transmission Control Protocol 传输控制协议

    TCP服务包括了面向连接服务和可靠数据传输服务。TCP提供了流量控制和拥塞控制。

    • Connection-oriented service 面向连接的服务：应用程序进程发送报文前通过TCP三次握手建立TCP连接，该连接是全双工的。结束报文发送时拆除该连接。
    • RDT Reliable Data Transfer 可靠数据传输：TCP的可靠数据传输确保一个进程从其接收缓存中读出非损坏（不错）、无间隔（不丢）、非冗余（不重）、按序（不乱）的数据流，即与发送的数据流完全一致。

    TCP连接提供full-duplex service全双工服务；TCP连接是point-to-point点对点的

    MSS Maximum Segment Size 最大报文段长：限制了TCP可以从缓存中取出并放入报文段中的数据量。MSS指的是报文段中应用层数据的最大长度，不包括TCP段头。

    MTU Maximum transmission unit 最大传输单元：最大链路层帧长度。其决定了MSS的长度。

    TCP报文段首格式

    TCP段头通常是20字节。包括源端口号、目的端口号、检验和字段；以及下面的

    • 序号字段：用于可靠数据传输。一个报文段的序号是该报文段数据首字节字节流编号。通常第一个数据流首字节序号为0.
    • 确认号字段：TCP是全双工的，因此主机A填充，发送给主机B的确认号是A期望从B那里收到的下一字节的序号。TCP只确认数据流中到第一个字节为止的字节（累计确认）。
    • 接收窗口：用于流量控制
    • 选项字段：可选，变长
    • 标志字段：
      • ACK: 用于指示确认字段中的值有效
      • RST、SYN、FIN：用于连接的建立和拆除
      • PSH：接收方应立刻将数据交给上层
      • URG：存在被上层置位紧急的数据

    由于超时间隔Time-out显然应当大于RTT，估计RTT就非常重要。

    TCP维持一个sampleRTT的均值EstimatedRTT=(1-a)EstimatedRTT + a(SampleRTT)。这个加权平均对最新样本的权重要大于老样本，因为越新的样本越能更好反映网络当前的拥塞情况。这被称作 EWMA Exponential Weighted Moving Average 指数加权移动平均

    DevRTT=(1-b)*DevRTT + b*|SampleRTT-EstimateRTT| 用于表示SampleRTT偏离EstimateRTT的程度

    重传超时间隔 TimeoutInverval = EstimateRTT + 4*DevRTT

    fast retransmit 快速重传 发送方在收到对相同数据的3个冗余ACK，便认为这个报文段后面的报文段已丢失，发送方便进行快速重传，在该报文段定时器过期前便重传丢失的报文段。（收到冗余ACK还可能是由于失序）

    可以认为TCP是GBN和SR的混合，因为大部分TCP实现会将失序但正确的segment缓存，并且TCP使用单一的重传计时器，每次重传1个报文段，配合累积确认机制，不会重传大量报文段。

    TCP的 flow control 流量控制

    流量控制用以消除发送方使接收方缓存溢出的可能。其使得发送方发送速率与接收方的读速相匹配。

    TCP通过让发送方维护receive window 接收窗口来提供流量控制。接收方计算RecBuffer - (LastRecv - LastRead) = RecWindow; 发送方维持 (LastSent - LastAck) <= RecWindow。

    当RecWindow = 0时，发送方依然需要发送只有一个字节数据的报文段，这些报文段会被接收方确认，直到缓存开始清空后RecWindow 不为0。

    TCP连接的三次握手过程
    • 客户端发出SYN=1, Seq=client_isn的TCP报文段
    • 服务器收到后，返回SYN=1, ACK=client_isn+1, Seq = server_isn的报文段（被称为SYNACK segment）
    • 客户端收到后，返回SYN=0, ACK=server_isn+1, Seq = client_isn+1的报文段

    服务器在TCP连接建立的第二次握手(SYNACK)时就初始化变量和缓存，如果有大量TCP不完成第三次握手的TCP SYN发送给服务器，就称作SYN洪泛攻击。

    TCP连接的关闭过程
    • 客户端发出关闭申请：FIN=1，服务器返回确认报文。
    • 服务器发送中止报文：FIN=1，客户端返回确认报文。
    TCP的Congestion Control 拥塞控制

    Congestion Control 拥塞控制 当发生网络拥塞时，TCP协议会试图限制每个TCP连接，使它们达到公平共享带宽的目的。

    注意：丢包的原因是网络拥塞，进而引发的路由器缓存溢出。

    当分组到达速率接近链路容量时，分组会经历巨大的排队延迟

    TCP让每一个发送方根据所感知到的网络拥塞的程度，来限制其能向连接发送流量的速率。如果一个TCP发送方感知从他到目的地之间的路径上没有拥塞，则增加速率，如果感知到拥塞，则降低发送速率。

    TCP拥塞控制机制让连接的每一段记录congestion window拥塞窗口，它对TCP发送方能向网络中发送流量的速率进行了限制。发送方的未确认数据量不超过拥塞窗口和接收窗口中的最小值。

    TCP发送方的丢包事件分为超时和收到3个冗余ACK，发送方检测到后就认为出现了拥塞。

    TCP拥塞控制算法包括三个部分：

    • AIMD Additive-Increase, Multiplicative-Decrease 加性增，乘性减

      每发生一次丢包，就将拥塞窗口减半；每收到一个ACK，就把拥塞窗口增大一个MSS。

      TCP拥塞控制协议的线性增长阶段被称为congestion avoidance避免拥塞。

    • slow start 慢启动

      TCP连接开始时，拥塞窗口大小为1个MSS；在初始阶段，不是线性，而是每过一个RTT将拥塞窗口翻一倍，直至发生第一个丢包事件。

    • 对超时做出反应

      收到3个冗余ACK时减半且线性增长；但超时事件发生时，进入慢启动阶段，将拥塞窗口设为1，指数增长，直到拥塞窗口达到超时事件前窗口值的一半（即阈值），此后拥塞窗口线性增长。

      TCP通过维持Threshold 阈值来管理这一过程。阈值初始化时设置为一个很大的值，每发生一个丢包事件，阈值便设置为当前拥塞窗口的一半，并维持到下一个丢包事件为止。

    在收到3个冗余ACK后取消慢启动阶段的行为称作fast recovery 快速恢复。

    SSL Secure Socket Layer 安全套接字层：提供了进程到进程的安全性服务，是对TCP的加强。

    网络层

    网络层的作用是将数据从一台发送主机移动到一台接收主机

    网络层的重要功能：

    • Forwarding 转发 分组到达某路由器的一条输入链路时。路由器将之移动到合适的输出链路
    • Routing选路 分组从发送方流向输出方是，网络层需要决定它们的路径。计算该路径的算法被称作routing algorithm选路算法。
    • 连接建立 虚电路网络中，网络层数据分组开始流动前在给定的源到目的地连接之间建立状态。

    因特网的网络层提供的服务称作Best-effort service尽力而为服务。

    ATM Asynchronous Transfer Mode 异步传输模式

    ATM网络的两个最重要的服务模型是恒定比特率服务和可用比特率服务：

    • CBR Constant bit rate 恒定比特率：一个信元的端到端时延、时延抖动、丢失、推迟交付的比例都可以确保在规定值以下。
    • ABR Available bit rate 可用比特率：MCR Minimum Cell transmission Rate最小信元传输速率 可以得到保证。ATM ABR 是一种基于速率的方法，即发送方明确地计算出它所能发送的最大速率，并据此对自己进行相应的调整。

    网络层的两种服务

    网络层提供了主机到主机的无连接和有连接的两种服务（并不是同时提供）

    虚电路

    VC Virtual Circuit 虚电路 在网络层提供的连接服务。ATM和帧中继网络都是虚电路网络。

    虚电路网络其组成如下：

    • 源和目的主机之间的路径（一系列链路和路由器）；
    • VC号，虚电路号，每段链路的一个号码
    • 沿着该路径的每台路由器中的转发表项

    虚电路网络中每台路由器的转发表包括了对VC号的转换。

    在虚电路网络中，该网络的路由器必须为进行中的连接维持连接状态信息connection state information。即在创建连接时将连接项添加至转发表、释放连接时从表中删除该项。

    沿两个端系统之间路径上的路由器都要参与虚电路的建立，且每台路由器都完全知道经过它的所有虚电路。

    虚电路有3个阶段：

    • 虚电路建立
    • 数据传输
    • 虚电路拆除。

    用于建立虚电路的报文被称作signaling message 信令报文，其协议称作signaling protocol 信令协议

    数据报网络

    datagram network数据报网络无需建立连接，路由器不维护连接状态信息。但是路由器在其转发表中维持了转发状态信息。路由器使用packet的目的地址来转发该分组，该过程使用longest prefix matching rule 最长前缀匹配规则。

    在数据报网络中，转发表由选路算法routing algorithm修改。

    路由器的工作原理

    每台路由器都有一张Forwarding Table 转发表

    路由器的4个组成部分：

    • Input ports 输入端口
    • Switching fabric 交换结构：将输入端口连接到输出端口
    • Output ports输出端口
    • Routing processor选路处理器：执行选路协议，维护选路信息与转发表，执行路由器中的网络管理功能

    line speed线路速度指执行一次查找的时间小于从输入端口接收一个分组所要的时间。从而使得对收到分组的处理可以在下一个接收操作结束之前完成。

    CAM Content Addressable Memory 内容可寻址内存：可以将IP地址提交，以常数时间返回对应转发表项内容。

    3种交换技术：经内存交换；经总线交换；经interconnection network 交换。

    丢包原因：网络拥塞使得路由器缓存空间被耗尽。

    switching fabric speed交换结构速率 交换结构将分组从输入端口移动到输出端口的速率

    AQM Active Queue Management 主动队列管理：在缓存填满前便丢弃或标记一个分组

    RED Random Early Detection 随机早期检测

    HOL Head-Of-the-Line 线路前部阻塞，即队首阻塞，即使输出端口可用，一个输入队列中排队的分组也必须等待通过交换结构发送，因为位于其队列前部的另一个分组阻塞。

    Internet Protocol (IP) 网际协议

    IP 网际协议 为主机之间提供逻辑通信功能，其服务模型为best-effort delivery service 尽力而为交付服务，但其不做任何确保，是unreliable service不可靠服务。

    IP是被分配给Interface的。

    网络层packet被称作 datagram 数据报

    因特网的网络层有三个组件：IP协议 IP Protocol，选路组件 routing component，互联网控制报文协议ICMP

    IP

    IPv4 datagram的格式如下

    • 版本号：v4就是4,v6就是6
    • Header Length 头部长度，其值x4等于首部字节数（一行4字节32位，这个值是行数）
    • **TOS Type Of Service **服务类型，用来区别不同类型的IP Datagram
    • datagram length：首部加数据的总长度
    • 标识，flags，片偏移：用于IPv4的分片fragment：分片的flag为为1，Identifier跟首片相同，最后一片flag是0，避免丢失用偏移字段指定片放在哪个位置。
    • TTL Time-To-Live 生存时间 ：用于确保数据报不会永远在网络内循环。每经过一个路由器，TTL-1；TTL为0的数据报会被路由器丢弃。
    • Protocol，上层协议
    • Header Checksum 首部校验和
    • 源和目的IP
    • 选项
    • 数据
    IPv4编址

    IPv4的地址长度为32位，用dotted-decimal notation 点分十进制表示

    独立的网络叫做一个subnet子网。

    子网的地址记作A.B.C.D/24 ,其中右侧部分叫做subnet mask子网掩码，

    IP地址中，子网网络号末尾为0；有限广播地址4个255；如果net-id是具体的网络号，而host-id是全1，就叫做定向广播(directed broadcast)，因为这是对某一个具体的网络（即net-id指明的网络）上的所有主机进行广播的一种地址；

    IP地址有A、B、C类（classful addressing分类编址）：

    • A类：1字节网络号。该类IP地址的最前面为“0”
    • B类：2字节网络号。最前面是10.
    • C类：3字节网络号。最前面是110.

    回送地址：127.0.0.1。

    CIDR Classless Interdomain Routing 无类别域间因特网的地址分配策略

    ICANN Internet Corporation for Assigned Names and Numbers 因特网名字与号码分配机构管理IP地址、DNS根服务器、分配域名、解决域名纠纷。

    IPv6

    头部长度固定40字节，源和目标地址均为128位，加入了flow流标签，不再允许分片（通过ICMP回送分组太大 的差错报文）。

    IPv4和IPv6的转换

    • dual-stack 双协议栈：

      将IPv6直接转为IPv4报文，会丢失诸如流标签等字段

    • tunneling 隧道

      将整个IPv6报文放到IPv4的数据字段中

    DHCP Dynamic Host Configuration Protocol 动态主机配置协议

    主机通过DHCP可以自动获取IP地址、子网掩码、默认网关、本地DNS服务器地址

    DHCP协议的步骤：

    1. DHCP 服务器发现：客户端发送DHCP discover message，源IP全0，目标IP全1
    2. DHCP 服务器提供：DHCP服务器收到后回复 DHCP offer message，源IP为自身，目标IP依旧是广播地址，报文中含有事务ID、推荐IP、子网掩码、IP地址租期 lease time等信息。
    3. DHCP 请求：客户端收到后，用DHCP request message对DHCP服务器进行响应，回显配置参数
    4. DHCP ACK：服务器用DHCP ACK message 对request进行响应确认。
    NAT Network Address Translation 网络地址翻译

    通过NAT转换表——记录了端口号和IP——来实现网络地址翻译。NAT能够使路由器对外界隐藏内部网络的细节。其将内网地址转为公网地址。但NAT使用了端口号，其本意是用来编址进程的。

    ICMP Internet Control Message Protocol 互联网控制报文协议

    ICMP最典型的用途是差错报告。

    Routing Algorithm 路由算法

    • 全局路由算法global routing algorithm: 用完整的、全局性的网络知识来计算从源到目标的最短路：LS链路状态算法
    • 分布式路由算法decentralized routing algorithm: 以迭代的分布式的方式计算出最短路。：DV距离向量算法

    当今的因特网选路算法都是负载不敏感的。

    LS Link-State Routing Algorithm ==链路状态路由算法==

    通过让每个节点向网络中的所有其他路由器广播link state packet链路状态包实现的，通过链路状态广播算法来完成。LS使用了Dijkstra 算法。

    通过为每个目的节点存放从起点到它的最短路上的下一跳节点，可以构建一个节点的转发表。

    OSPF Open Shortest Path First 开放最短路径优先使用了LS链路状态算法

    DV Distance-Vector ==距离向量算法==

    DV是一个迭代的、异步的、分布式的算法。其采用Bellman-Ford方程。

    每个节点维护一张routing table路由表。可能遇到选路环路问题。

    RIP Routing Information Protocol 选路信息协议使用了DV距离向量算法

    AS Autonomous system 自治系统 由一组通常在相同管理控制下（比如同一个ISP）的路由器组成。同一个AS内的路由器全部运行同样的路由算法。

    • Intra-AS Routing protocol自治系统内部选路协议：又称interior gateway protocol内部网关协议，指在一个自治系统内运行的选路算法。

      RIP与OSPF均为自治系统内部选路协议

    • Inter-AS Routing protocol自治系统间选路协议：用于从相邻AS获取可达性信息以及向AS内的所有路由器传播可达性信息。负责这个功能的路由器叫做gateway router网关路由器（所以一台Gateway同时要有BGP和RIP/OSPF）

      BGP Broder Gateway Protocol 边界网关协议 是当今因特网中域间选路协议的标准。其从相邻AS获得可达性信息、向AS内所有路由器传播这些信息、基于可达性信息和AS策略决定到达子网的好路由。
